## Structure

### H2 : Qu'est-ce qu'une application mobile — et pourquoi c'est différent d'un site ?
**Keywords cibles :** développer une application mobile, application iOS et Android, PWA
**Contenu attendu :** Définition rapide, sans condescendance : application native (iOS/Android), application hybride (Flutter, React Native), PWA. Mettre en relief la vraie différence avec un site web responsive : accès aux capteurs hardware (GPS, appareil photo, notifications push), fonctionnement hors-ligne, distribution via les stores. 2-3 paragraphes max. Pas de liste à puces creuse — chaque type doit être illustré par un exemple de cas d'usage concret (ex : artisan qui scanne un QR code sur chantier → app native ; site vitrine PME qu'on consulte depuis mobile → pas besoin d'app).
**Lien sortant :** → `/blog/application-mobile-ou-pwa` (satellite à créer — ancre : "app mobile ou PWA")

### H2 : Quand une PME romande a vraiment besoin d'une application mobile
**Keywords cibles :** application mobile pour PME en Suisse romande, créer une application mobile
**Contenu attendu :** 4 critères décisionnels clairs (fréquence d'utilisation élevée, besoin d'accès hors-ligne, accès aux capteurs hardware, besoin de notifications push géolocalisées). Contra-exemples honnêtes : quand une PWA ou un site responsive suffit et économise 20 000 CHF. Tonalité : Jon parle d'abord aux cas où l'app est pertinente, mais ne cache pas les cas où c'est du sur-vente. Exemples sectoriels : artisan BTP (suivi chantier hors-ligne), PME avec équipe terrain, commerce avec programme de fidélité.
**Lien sortant :** → `/blog/application-mobile-pme-suisse` (satellite à créer — ancre : "vaut-il le coup pour votre PME")

### H2 : Les étapes d'un projet d'application mobile — de l'idée au lancement
**Keywords cibles :** développement d'une application mobile en Suisse, créer une application mobile
**Contenu attendu :** Pipeline de 5 étapes concrètes : 1. Cadrage / discovery (1-2 sem.) → 2. UX / wireframes (1-2 sem.) → 3. Développement (6-16 sem. selon complexité) → 4. Tests & QA (2-3 sem.) → 5. Publication stores + suivi post-lancement. Pour chaque étape : ce qui se passe, qui fait quoi (client vs prestataire), et à quel moment le client doit être disponible. Insister sur le point souvent oublié : la publication sur l'App Store Apple nécessite un compte développeur (99 USD/an) et un délai de validation de 1-7 jours. Ce sont des infos concrètes que personne ne met en avant.

Recommandation Jon Labs : pour une PME avec budget modéré, Flutter ou React Native livrent 80% des résultats d'une app native à 40-50% du coût. Mention que Jon travaille avec Flutter et React Native.
