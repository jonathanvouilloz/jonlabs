## Structure cible (H2/H3)

### Introduction
Fait concret d'ouverture : Pinterest a converti son site mobile en PWA et a vu l'engagement augmenter de 60%. Starbucks a une PWA qui fonctionne offline pour prendre des commandes sans connexion. Pourtant, quand une PME romande pense "app", elle pense automatiquement App Store. Ce réflexe coûte cher.

### H2 : Définitions en 2 lignes

**Application mobile native** : installée depuis l'App Store (Apple) ou Play Store (Google), accède directement aux fonctionnalités hardware du téléphone, fonctionne offline selon le dev.

**Progressive Web App (PWA)** : un site web avec des superpouvoirs — installable sur l'écran d'accueil, fonctionnement offline possible via cache, notifications push (selon les limites des navigateurs). Pas de stores requis.

### H2 : Ce que la PWA ne peut pas faire (limites réelles)
- Notifications push iOS : Apple a longtemps bloqué — depuis iOS 16.4, disponible mais avec contraintes (user doit installer la PWA d'abord)
- Bluetooth, NFC, USB avancé : accès limité ou nul selon navigateur
- Paiements in-app via Apple Pay / Google Pay : complexe à intégrer hors store
- Publication sur l'App Store / Play Store : impossible sans encapsuler la PWA (solutions comme Capacitor)
- Reconnaissance faciale via Face ID : API limitée sur PWA

### H2 : Ce que la PWA fait mieux (et que tu sous-estimes)
- **Coût** : une PWA coûte 2 à 5× moins cher qu'une app native (pas de 2 codebases iOS/Android)
- **SEO** : une PWA est indexable par Google — une app native ne l'est pas
- **Mises à jour** : instantanées, sans validation App Store (qui prend 1-3 jours)
- **Cross-platform** : fonctionne sur iOS, Android, desktop — sans refaire le travail
- **Pas de friction d'installation** : l'utilisateur n'a pas à passer par un store

### H2 : Tableau de décision — App native ou PWA ?

| Critère | → App native | → PWA |
|---|---|---|
| Notifications push iOS (critiques) | ✓ | Limité |
| Accès Bluetooth / NFC | ✓ | ✗ |
| Fonctionnement offline complet | ✓ | Partiel |
| Présence sur App Store souhaitée | ✓ | ✗ |
| Budget < 15 000 CHF | ✗ | ✓ |
| SEO important | ✗ | ✓ |
| Mises à jour fréquentes | ✗ | ✓ |
| Première version / test de concept | ✗ | ✓ |
| Accès caméra, GPS | ✓ | Partiel (selon navigateur) |

### H2 : Exemples de cas d'usage PME

**PWA adaptée :**
- Application de prise de rendez-vous pour un cabinet médical → navigateur suffit
- Catalogue produits avec recherche offline pour un représentant commercial → PWA parfaite
- Outil de suivi de présence pour une association → web app responsive

**App native nécessaire :**
- App de livraison avec scan QR, notifications temps réel et mode offline complet → app native requise
- App de chantier avec photos géolocalisées et rapport offline → app native
- App de fidélité avec paiements in-app → app native pour les stores

### H2 : La règle des 3 questions

Avant de choisir, réponds honnêtement :
1. As-tu besoin d'un accès hardware avancé (Bluetooth, NFC) ou de notifications push critiques sur iOS ?
2. Ton budget est-il inférieur à 20 000 CHF ?
3. Veux-tu apparaître sur Google (SEO) ?

Si tu réponds "non / oui / oui" → commence par une PWA. Si tu réponds "oui" à la question 1 → app native.
